# System Design

확장성, 캐시, 메시지 큐, 장애 격리, 관측 가능성과 관련된 면접 질문을 정리합니다.

## 질문 목록

- scale-up과 scale-out의 차이는 무엇인가요?
- 메시지 큐를 왜 사용하나요?
- 동기 처리와 비동기 처리의 장단점은 무엇인가요?
- 멱등성이란 무엇인가요?
- 외부 API 장애를 어떻게 격리할 수 있나요?
- p95, p99 latency를 왜 보나요?
- 로그, 메트릭, 트레이스의 차이는 무엇인가요?
- 단일 장애 지점(SPOF)이란 무엇인가요?
- 동기 방식으로 외부 서비스를 호출할 때 외부 서비스 장애가 나면 어떻게 조치할 수 있나요?
- 로드 밸런싱에 대해 설명해주세요.
- 분산 트랜잭션 처리 방식인 2PC, Saga, Transactional Outbox를 설명해주세요.
- 분산 환경에서 Redis를 활용한 잠금은 어떻게 구현할 수 있나요?
- 스케일 아웃과 스케일 업의 차이점을 설명해주세요.
- 시스템 간 비동기 연동 방식에는 무엇이 있나요?
- 이벤트 소싱이란 무엇인가요?
- 최종적 일관성이란 무엇인가요?
- 캐시 스탬피드 현상에 대하여 설명해주세요.
- 캐싱 전략에 대해 설명해주세요.
- 트랜잭셔널 아웃박스 패턴에 대해 설명해주세요.
- CAP 정리에 대해서 알고 계신가요?
- CQRS 패턴이란 무엇인가요?
- DB Replication에 대해 설명해주세요.
- Graceful Shutdown의 필요성에 대해 설명해주세요.
- HTTP 헤더에서 캐시, 인증, 콘텐츠 협상, 보안 관련 헤더를 설명해주세요.
- Redis가 싱글 스레드로 만들어진 이유를 설명해주세요.
